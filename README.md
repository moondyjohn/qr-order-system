# 二维码扫码点餐系统

## 功能特性

### 1. 生成二维码台号
- 在「管理后台 → 台号管理」中添加台号（如 A01、1号桌、包厢1）
- 系统自动生成该台号的专属点餐二维码
- 支持下载二维码图片、复制点餐链接
- 顾客扫码即可进入该台号的点餐页面

### 2. 台号点餐界面
- 扫码后自动识别台号
- 按分类浏览菜品（热菜/凉菜/汤品/饮品/主食/甜品）
- 点击 +/- 按钮快速选择数量
- 实时显示已选数量和总价
- 支持添加备注（如：少辣、不要香菜）
- 底部购物车栏一键提交

### 3. 管理后台
- **菜品管理**：添加/编辑/删除菜品，设置名称、分类、价格、库存、描述、图片、上架状态
- **台号管理**：添加/删除台号，查看/下载二维码
- **订单管理**：查看所有订单，按状态筛选，更新订单状态（待处理→已确认→已完成）

## 快速启动

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动系统
python app.py

# 3. 访问系统
# 顾客点餐首页： http://localhost:5000
# 管理后台：     http://localhost:5000/admin
```

## 使用流程

### 商家设置（首次使用）
1. 访问 `http://localhost:5000/admin`
2. 进入「台号管理」→ 添加所有台号（如 A01~A20）
3. 进入「菜品管理」→ 添加所有菜品，上传图片
4. 打印台号二维码，贴在对应桌子上

### 顾客点餐
1. 扫描桌上的二维码
2. 浏览菜单，点击 + 添加菜品
3. 点击底部「提交订单」
4. 可选：填写备注（口味偏好等）

### 商家接单
1. 在「订单管理」查看新订单（状态：待处理）
2. 点击订单查看详情，确认接单 → 状态变为「已确认」
3. 出餐后点击「完成订单」

## 数据库结构

| 表名 | 说明 |
|------|------|
| tables | 台号表（台号名称、二维码路径） |
| categories | 分类表（热菜、凉菜等） |
| dishes | 菜品表（名称、价格、库存、图片等） |
| orders | 订单表（台号、状态、总价、备注） |
| order_items | 订单明细表（菜品、数量、单价） |

## 默认分类

系统自动创建以下分类：热菜、凉菜、汤品、饮品、主食、甜品

## 技术栈

- **后端**：Python Flask + SQLAlchemy（SQLite）
- **前端**：HTML + CSS + 原生 JavaScript
- **二维码**：qrcode (Pillow)
- **图片上传**：Werkzeug

## 目录结构

```
qr-order-system/
├── app.py                  # 主应用（路由 + API）
├── database.py             # 数据库模型定义
├── requirements.txt        # 依赖清单
├── qr_order.db            # SQLite 数据库（自动生成）
├── static/
│   ├── css/style.css     # 全局样式
│   ├── uploads/           # 菜品图片上传目录
│   └── qrcodes/          # 二维码图片目录
└── templates/
    ├── base.html          # 基础模板
    ├── index.html         # 首页（选择台号）
    ├── table_order.html   # 点餐页面
    ├── admin.html         # 管理后台布局
    ├── admin_dishes.html  # 菜品管理
    ├── admin_tables.html  # 台号管理
    └── admin_orders.html  # 订单管理
```

## 注意事项

- 生产环境请修改 `app.py` 中的 `SECRET_KEY`
- 如需外网访问，将 `app.run()` 中的 `host` 改为服务器 IP，并配置公网访问
- 二维码中的 URL 包含当前访问地址，部署时请确保地址正确
- 建议定期备份 `qr_order.db` 数据库文件
